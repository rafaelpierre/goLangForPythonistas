The user is a Python engineer learning Go. They have enough Python fluency that Python analogies are a useful teaching tool (contrast duck typing vs interfaces, asyncio vs goroutines, exceptions vs error values, list vs slice, dict vs map, dataclass vs struct, etc.).

Go experience level at session start: beginner (had only a Hello World file in crashCourse/intro/01_hello.go).

Learning goal: gain practical, idiomatic Go skills -- not just syntax familiarity.

Preferred exercise style: hands-on, runnable single-file exercises with TODO markers, organized by topic in subfolders under crashCourse/.
